# 儿童积分管理系统 需求文档（冻结版）

## 1. 文档目的

本文档用于固化当前阶段已确认需求，作为后续 OpenSpec proposal/spec 与实现验收依据。

## 2. 产品目标

构建家庭内部使用的儿童积分管理 Web 应用，支持高频记录行为加减分，确保历史可信可追溯，并通过月视图帮助孩子和家长理解行为趋势。

## 3. 适用范围

- 面向家庭内部场景
- 前端静态部署
- 多设备同步
- 在线数据存储与权限控制

## 4. 用户角色与权限

### 4.1 角色定义

- parent（家长）
- child（孩子）

### 4.2 登录与账号策略

- 系统必须登录后访问，禁止匿名访问。
- 不开放注册页面和自助注册接口。
- 账号由管理员在数据库后台手动创建。

### 4.3 权限矩阵

- parent 可执行：
  - 孩子管理
  - 规则管理（新增、编辑、停用）
  - 积分记录新增
  - 查看全部孩子数据
- child 可执行：
  - 仅查看自己数据（总分、历史、月视图）
- child 不可执行：
  - 任何写入操作

## 5. 安全要求

### 5.1 密码与认证

- 密码只存储不可逆哈希，禁止明文与可解密密文。
- 使用哈希加盐机制（每用户独立 salt）。
- 推荐算法：Argon2id（优先）或 bcrypt。
- 可选增强：服务端 pepper（仅环境变量存储，不入库）。

### 5.2 访问控制

- 所有业务数据读取和写入均需基于登录态与角色校验。
- child 账号仅可读取绑定 child_id 的数据。
- 历史记录接口禁止更新与删除。

### 5.3 账号治理

- 用户表必须包含 is_active 字段，用于禁用账号。
- 建议启用登录失败限流（例如连续失败锁定）。

## 6. 功能需求

## 6.1 孩子管理

- 支持新增孩子。
- 支持隐藏孩子（逻辑隐藏），不删除历史数据。
- 每个孩子拥有独立积分统计。

## 6.2 积分规则管理

### 6.2.1 规则类型

- 默认规则（系统初始化提供）
- 自定义规则（家长新增）

### 6.2.2 规则生命周期与编辑

- 默认规则允许编辑名称和分值。
- 默认规则不允许删除，仅允许停用。
- 自定义规则允许编辑和停用。
- 规则编辑仅影响未来记录，不回溯历史。

### 6.2.3 规则编辑页面

- 系统必须提供规则编辑页面。
- 页面至少支持：
  - 修改规则名称
  - 修改规则分值（正/负）
  - 切换启用/停用状态
  - 标识默认规则与自定义规则

## 6.3 积分记录

### 6.3.1 记录流程

- 选择孩子
- 选择日期（默认当天）
- 选择一个或多个规则
- 自动计算本次总分变化
- 记分时不允许手动填写分数
- 提交

### 6.3.2 入库规则

- 前端可多选规则，但入库必须拆分为多条记录（1 规则 = 1 记录）。
- 每条记录的分值必须来源于所选规则，不接受手动输入分值。
- 每条记录必须不可变。

### 6.3.3 纠错机制

- 不允许编辑历史记录。
- 不允许删除历史记录。
- 错误只能通过新增反向记录冲正。

## 6.4 总分统计

- 每个孩子总分为历史记录累加值。
- 新增记录后总分应实时更新。

## 6.5 月视图看板（第一阶段）

### 6.5.1 核心要求

- 提供月历视图，展示当月每日净积分（正/负/零）。
- 支持翻看历史月份和未来月份。
- 支持切换孩子查看对应月数据（child 仅看自己）。

### 6.5.2 交互要求

- 顶部显示当前年月。
- 提供上月/下月切换按钮。
- 点击某天进入日详情，查看当天全部记录。

### 6.5.3 视觉语义

- 正分使用绿色语义。
- 负分使用红色语义。
- 零分使用中性语义。

## 6.6 历史列表与日详情

- 历史列表按时间倒序展示。
- 日详情展示该日全部记录，至少包含：
  - 规则名称快照
  - 分值快照
  - 创建时间

## 7. 数据模型要求

## 7.1 Child

- id
- name
- is_active（可选，推荐）
- created_at

## 7.2 Rule

- id
- name
- score
- is_default
- is_active
- created_at
- updated_at

## 7.3 Record（不可变事件）

- id
- child_id
- rule_id（可空，保留兼容）
- rule_name_snapshot
- score_snapshot
- date（按天）
- created_at
- created_by
- correction_of（可空，指向被冲正记录）

## 7.4 User

- id
- username 或 email
- password_hash
- role（parent/child）
- child_id（child 必填，parent 为空）
- is_active
- created_at
- updated_at

## 8. 业务约束

- 历史记录不可修改、不可删除。
- 规则变更不影响历史记录。
- 任何积分变化必须可追溯到具体记录。
- 多设备看到一致数据结果。

## 9. MVP 范围（第一阶段）

- 登录认证（无注册，后台建号）
- 角色权限（parent/child）
- 孩子管理
- 规则管理（编辑与停用）
- 快速记分（多选规则、拆单入库）
- 不支持在新增记录时手动输入自定义分值
- 总分与历史列表
- 月视图看板（可翻月）

## 10. 非目标（当前阶段不做）

- 公开注册
- 社交/分享能力
- 排行榜与成就系统
- 高级统计图表
- PWA 离线增强

## 11. 验收标准

### 11.1 安全与权限

- 未登录访问任何业务页面时被拦截。
- child 账号无法访问或写入他人数据。
- 数据库中不存在明文密码字段。

### 11.2 规则与历史可信

- 规则编辑后，历史记录显示原始快照不变。
- 默认规则无法删除，但可停用。
- 任一历史记录无法通过 UI/API 修改或删除。

### 11.3 记分与统计

- 单次多规则记分可成功写入多条记录。
- 记分界面不存在手动输入分数字段。
- 提交时即使请求带有自定义分值，也会按规则分值写入。
- 总分与历史累加一致。
- 反向冲正后，总分与明细可解释。

### 11.4 月视图

- 可切换上月/下月并正确展示对应月份数据。
- 每日净积分展示正确，颜色语义正确。
- 点击日期可进入当日明细。

## 12. 待确认项（实现前补齐）

- PIN 功能是否与账号密码并存，或移除 PIN。
- 登录失败锁定的具体阈值和时长。
- 未来日期是否允许记分。
- 备注字段是否纳入第一阶段。

## 13. 第二阶段需求（已确认）

- 新增记录支持“临时规则”快捷创建。
- 临时规则由家长输入规则名称和固定分值后创建，再参与勾选记分。
- 临时规则创建后进入规则库，后续可复用，并遵循规则编辑与停用机制。
- 第二阶段仍不支持绕过规则直接手动填写单条记录分值。
