# Feature Specification: 儿童积分管理系统（Family Score System）

**Feature Branch**: `001-family-score-system`  
**Created**: 2026-03-20  
**Status**: Draft  
**Input**: 基于 docs/REQUIREMENTS.md 澄清文档生成

## Overview

构建家庭内部使用的儿童积分管理 Web 应用，支持高频行为加减分记录，通过月视图帮助家长和孩子理解行为趋势，确保历史数据可信可追溯。

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 家长为孩子快速记分 (Priority: P1)

家长每天多次为孩子的行为加减分。家长登录后，进入月视图，点击今天，展开当天记录页，勾选一条或多条规则后，分数自动累加并在提交后写入历史。

**Why this priority**: 这是系统最核心的操作闭环，没有记分其他功能均无意义。

**Independent Test**: 无需其他功能，仅实现登录 + 孩子管理 + 规则管理 + 记分入库即可验证完整记分闭环。

**Acceptance Scenarios**:

1. **Given** 家长已登录，**When** 在日详情页面勾选"按时完成作业"并提交，**Then** 产生一条 score=+2 的不可变记录，当天总分和累计总分同步更新。
2. **Given** 家长同时勾选 2 条规则（+2 和 +3），**When** 提交，**Then** 入库 2 条独立记录，当天净分显示 +5。
3. **Given** 家长误操作，**When** 在同一天新增一条同规则的反向记录（score=-2），**Then** 历史中出现两条记录，净分归零，原始记录未被修改。
4. **Given** child 角色用户登录，**When** 尝试访问新增记录功能，**Then** 被拒绝，界面只显示只读内容。

---

### User Story 2 - 家长和孩子查看月视图看板 (Priority: P1)

家长和孩子均能打开月视图，看到每天的积分变化，翻看过去的月份，理解行为趋势。

**Why this priority**: 月视图是产品"可感知价值"的核心载体，孩子依赖它理解自己的趋势。

**Independent Test**: 可在有历史数据后独立验证：切换月份、颜色展示、点击日期进详情。

**Acceptance Scenarios**:

1. **Given** 当月有若干记录，**When** 打开月视图，**Then** 每一天正确显示当日净积分，正分绿色、负分红色、空日无标注。
2. **Given** 当前月是 2026-03，**When** 点击"上月"，**Then** 展示 2026-02 的数据，可继续往前翻。
3. **Given** child 账号登录，**When** 打开月视图，**Then** 只能看到自己 child_id 对应的数据。
4. **Given** 某天有记录，**When** 点击该日格子，**Then** 进入日详情页，展示当日全部记录（规则名快照、分值快照、时间）。

---

### User Story 3 - 家长管理规则 (Priority: P2)

家长可以查看、编辑、停用规则，也可以新增自定义规则。默认规则不可删除，仅可停用。

**Why this priority**: 规则管理是记分的前提，但系统预置了默认规则，可在记分功能后补充完善。

**Independent Test**: 独立打开规则管理页，完成"编辑名称/分值"、"停用默认规则"、"新增自定义规则"三个动作即可验证。

**Acceptance Scenarios**:

1. **Given** 家长进入规则管理，**When** 修改"按时完成作业"分值为 +3，**Then** 新记录使用 +3，历史记录仍显示原快照 +2。
2. **Given** 家长尝试删除默认规则，**When** 点击删除，**Then** 操作不可用（按钮禁用或无删除入口）。
3. **Given** 家长停用一条规则，**When** 打开新增记录面板，**Then** 该规则不出现在可勾选列表中。
4. **Given** 家长新增自定义规则"游泳练习（+4）"，**When** 保存，**Then** 规则立即出现在记分列表中。

---

### User Story 4 - 登录与访问控制 (Priority: P2)

未登录用户无法访问任何业务页面；parent 账号可写入；child 账号只读自己数据。

**Why this priority**: 认证是安全基线，但可在核心功能稳定后集成验收。

**Independent Test**: 可单独创建两类账号，验证未登录跳转、parent 写入、child 只读三条路径。

**Acceptance Scenarios**:

1. **Given** 未登录用户，**When** 访问任何业务路径，**Then** 被重定向到登录页。
2. **Given** child 账号已登录，**When** 尝试调用新增记录接口，**Then** 返回权限拒绝，前端不显示写入入口。
3. **Given** 连续输错密码 5 次，**When** 第 6 次尝试，**Then** 账号临时锁定，提示等待时间。
4. **Given** parent 将某账号 is_active 设为 false，**When** 该用户尝试登录，**Then** 提示账号已停用。

---

### User Story 5 - 家长管理孩子档案 (Priority: P3)

家长可以新增孩子、隐藏（软删除）孩子，每个孩子有独立积分。

**Why this priority**: 属于配置项，建立孩子记录后系统才能正常运作，但逻辑简单。

**Independent Test**: 新增孩子后，进入记分流程选择该孩子，可完整验证。

**Acceptance Scenarios**:

1. **Given** 家长新增孩子"小明"，**When** 进入记分，**Then** 孩子选择器中出现"小明"。
2. **Given** 家长隐藏孩子"小明"，**When** 查看孩子列表，**Then** "小明"不显示，但历史记录保留完整。

---

### Edge Cases

- 某天没有任何记录，月视图该日格子如何显示（空白/0/无数值）？
- 家长为过去日期记分时，是否有任何限制？
- 两台设备同时提交记录，数据一致性如何保障（依赖后端事务）？
- 规则被停用后，历史含该规则的记录是否仍正常展示？（必须可显示，快照已冗余）
- 总分出现负数是否合法？（是，无下限）

## Clarifications

### Session 2026-03-20

- Q: 纠错（冲正）如何从 UI 触发？ → A: 家长通过标准新增记录流程手动操作，无专用"撤销/冲正"按钮，`correction_of` 字段不由 UI 自动填充。
- Q: 登录会话时长策略是什么？ → A: 永不过期，除非手动登出或账号被管理员停用（is_active=false）。
- Q: parent 在月视图中如何切换孩子？ → A: 首页展示所有孩子的卡片，点击某个孩子卡片进入该孩子的月视图；child 账号登录后直接进入自己的月视图，无切换入口。
- Q: 记分操作是否需要快捷入口？ → A: 月视图页提供"快速记分"浮动按钮（FAB），点击直接跳转到今日日详情页并自动展开记分面板，跳过手动点击日格子的步骤。
- Q: 系统初始化默认规则集内容？ → A: 空规则集，系统初始不提供任何预设规则，家长自行在规则管理页添加所有规则。

### Functional Requirements

#### 认证与账号

- **FR-001**: 系统必须要求登录后才能访问任何业务页面，禁止匿名访问。
- **FR-002**: 系统不提供注册页面或自助注册接口，账号由管理员在数据库中手动创建。
- **FR-003**: 密码必须以不可逆哈希存储，推荐 Argon2id 加独立 salt，禁止明文或可解密密文。
- **FR-004**: 连续登录失败达到阈值后，系统必须临时锁定账号。
- **FR-005**: 用户表必须支持停用（is_active 字段），停用账号无法登录。
- **FR-005b**: 登录会话永不自动过期；会话失效只能通过以下方式触发：用户手动登出、账号被管理员设为停用（is_active=false）。

#### 角色与权限

- **FR-006**: 系统必须支持 parent 和 child 两种角色。
- **FR-007**: parent 可执行：孩子管理、规则管理、积分记录新增、查看所有孩子数据。
- **FR-008**: child 只能读取绑定 child_id 的孩子的数据，不可执行任何写入操作。
- **FR-009**: 所有写入接口必须在服务端校验角色，拒绝 child 的写入请求。

#### 孩子管理

- **FR-010**: 家长必须能新增孩子（姓名必填）。
- **FR-011**: 家长必须能隐藏孩子（逻辑停用），历史记录不删除。
- **FR-012**: 每个孩子拥有独立的积分体系，不同孩子数据不互混。

#### 积分规则管理

- **FR-013**: 系统初始化时规则库为空，不提供任何预设规则；家长必须通过规则管理页自行添加所有规则后才能开始记分。
- **FR-014**: 所有规则均由家长创建，均为自定义规则；不存在"默认规则不可删除"的限制，所有规则可编辑、停用或删除。
- **FR-015**: 家长可新增、编辑、停用自定义规则。
- **FR-016**: 规则编辑仅影响未来记录，不回溯历史（历史记录含规则名称和分值快照）。
- **FR-017**: 停用的规则不出现在新增记录的可选列表中。

#### 积分记录

- **FR-018**: 家长必须能选择孩子、选择日期（默认当天）、勾选一个或多个规则后提交记录。
- **FR-018b**: 月视图页必须提供"快速记分"浮动操作按钮（FAB），点击后直接进入今日日详情页并自动展开记分面板；此为高频操作的最短路径（2 步内可达）。
- **FR-019**: 记分时不允许手动填写分数，分数必须来源于所选规则定义。
- **FR-020**: 多选规则提交时，每条规则入库为独立不可变记录（1 规则 = 1 条 Record）。
- **FR-021**: 历史记录不允许修改或删除，纠错只能通过新增记录完成（选择与错误记录相反分值的规则）。
- **FR-022**: 每条记录必须冗余存储规则名称快照（rule_name_snapshot）和分值快照（score_snapshot）。UI 不提供专用"撤销/冲正"按钮，`correction_of` 字段预留供未来使用，第一阶段不由 UI 填充。
- **FR-023**: 冲正记录必须可关联原记录（correction_of 字段）。

#### 总分统计

- **FR-024**: 每个孩子总分为历史所有记录 score_snapshot 的累加，支持负数，无下限。
- **FR-025**: 新增记录后，总分和当日净分必须实时更新。

#### 月视图看板

- **FR-026**: 系统必须提供月历视图，展示当月每日净积分。登录后：parent 先进入孩子卡片首页，选择孩子后进入该孩子的月视图；child 登录后直接进入自己的月视图。
- **FR-027**: 月视图必须支持向前和向后翻月，无翻月次数限制。
- **FR-028**: 有记录的日期显示当日净分，正分绿色、负分红色，无记录日期显示为空。
- **FR-029**: 点击日期必须进入日详情页，展示当日全部记录（规则名快照、分值快照、时间）。
- **FR-030**: 月视图顶部必须显示当前年月，并提供上月/下月切换控件。

#### 安全约束

- **FR-031**: 历史记录接口在服务端必须拒绝 UPDATE 和 DELETE 操作。
- **FR-032**: child 账号的数据查询必须在服务端过滤，仅返回绑定 child_id 的数据。

---

### Key Entities

- **User**: 系统账号，含角色（parent/child）、密码哈希、关联孩子ID（child 必填）、停用状态。
- **Child**: 孩子档案，含姓名、停用状态，拥有独立积分空间。
- **Rule**: 积分规则，含规则名、固定分值（正/负）、是否默认、启用/停用状态。修改只影响未来。
- **Record**: 不可变积分事件，一规则一记录，含规则名快照、分值快照、日期、创建者、可关联冲正原记录。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 家长从登录到完成一次记录的操作步骤不超过 4 步，全程不超过 30 秒。
- **SC-002**: 月视图翻月响应时间在有网络连接的情况下不超过 1 秒。
- **SC-003**: 提交记录后，总分和月视图当日净分在 1 秒内完成刷新。
- **SC-004**: 100% 的历史记录在规则被修改或停用后仍能正确展示原始快照内容。
- **SC-005**: child 账号无论通过 UI 还是直接 API 调用均无法写入或修改任何数据。
- **SC-006**: 数据库不存在任何明文密码字段，所有密码均不可逆哈希存储。
- **SC-007**: 多设备（如手机和平板）打开同一孩子月视图，数据展示一致。

---

## Assumptions

- 初期家庭规模：1-2 位家长账号，2-4 个孩子账号，每天记录 5-20 条。
- 不需要为孩子提供密码重置功能，由家长在数据库侧协助处理。
- 系统时区按家庭所在地本地时区，前端负责展示本地日期，后端存 UTC。
- 未来日期允许查看但不允许新增记录（防止预写）。
- 备注功能延至第二阶段，第一阶段记录无需备注字段。

---

## Out of Scope (Phase 1)

- 公开注册与自助账号管理
- 新增记录时快捷创建临时规则（第二阶段）
- 备注功能
- 排行榜、成就系统、趋势统计图
- PWA 离线支持
